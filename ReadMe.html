PlayaPlaya

Introduction

This is a simple audiobook player. Features include:

  • Focused on audiobooks
  • Tracks the last playback position, and resumes when restarted.
  • Playlist-centric, where each playlist represents a book
  • Remembers positions on a per-playlist basis: open a playlist, and it'll resume where it left off.
  • Start it without specifying a playlist, and it'll resume with the last playlist, at the last position
  • Doesn't try to manage your audio files.
  • Doesn't try to manage your devices.
  • GStreamer-based audio playback

Limitations

  • Currently only supports .m3u playlists, and .mp3 files. This will change soon.
  • Installation is very easy, but it currently lacks the niceties like menu entries and mimetype associations.

YOUR contributions to fix these issues are very welcome. Otherwise, leave it with us and we'll get to it in good time.

Requirements

To run PlayaPlaya, you currently require a Unix-like operating system, such as Linux or BSD. PlayaPlaya may run on Mac and Windows, but this hasn't been tested yet. Beyond Unix, you only need the following:
  • PyGTK
  • PyGST

Your distro should provide all of these; please check your distro's documentation / support if you need help with installing them.

Installation

Installation is very easy; unzip the archive to any directory, and make a symlink from the playaplaya file within, to your bin directory.

For example, to install at /usr/local/soft/PlayaPlaya:

mkdir /usr/local/soft cd /usr/local/soft unzip x path_to_zip_file_v1.x.zip cd /usr/local/bin ln -s ../soft/PlayaPlaya_v1.x/playaplaya .

Be sure to replace v1.x with the appropriate version number, depending on which version you're installing.

If you've completed this successfully, PlayaPlaya will be available as "playaplaya" on the command line. You can also right-click a playlist on your desktop, and navigate to this command in /usr/local/bin, to have your desktop open playlists with PlayaPlaya from now on.

Usage

To use PlayaPlaya, simply run it, and open a .m3u playlist. The rest should be obvious.
